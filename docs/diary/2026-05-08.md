### Rozhodnutí

- **Stage 1 scope:** FixedJoint API + audio (click + spawn) + edge mask vizualizace + sleep-mode fix. **Stage 2 (α/β/γ) odloženo** na další sezení — rapier joint solver po tuningu **funguje**, takže hybrid orchestrace má jasný cíl: kompenzovat **gravity-side** dissipaci, ne joint-side.
- **`World.defaultCanSleep = false`** jako Pixelodynamics-wide default. Sleep mode je herní optimalizace, ve fyzikálním sandboxu **anti-feature** — uspí rotující rigid pair a tichne dynamics. Žádná regression na E1–E7 (manuální mode nebo bodies bez constraint nikdy nebudily sleep threshold tak agresivně).
- **Edge mask vizualizace:** 4-bit mask per pixel jako WebGL per-instance attribute. Maskovaná hrana se kreslí červeně (`#d86f6f`), ne discard. Důvody:
  - Plný discard rohových fragmentů (corner kde v_local ≈ (±0.5, ±0.5)) vytvořil černé „zobáčky" — corner má dPosX==dPosY==minDist, jakákoli z nich masked → discard celý roh.
  - 50% průhlednost (mezikrok) byl měkčí, ale uživatel preferuje explicitní vizuální signál (červená).
  - Long-distance jointy (anchor mimo obě tělesa) vytvářely U-shapes při full discard; červená barva je vizuální hint na spoj bez geometrie destrukce.
- **Joint orientation snap fix:** `JointData.fixed(anchorA, r_B − r_A, anchorB, 0)` místo `(0, 0)`. Frame parametry kódují **relativní rotaci constrainu**: `r_A + frame1 == r_B + frame2`. S nulami bylo `r_A == r_B`, což pro LMB-spawn pixely s random rotacemi violation → solver instant impulse → snap. Fix preserve aktuální rotation gap.
- **Naming:** zvuk pro joint create i remove je stejný (`click.mp3`) — symetrie eventů. Spawn LMB má vlastní zvuk (`spawn.mp3`).
- **E3 preset finální podoba:** canSleep=false + solverIters=16 + pgsIterations=4 (tuned). Drift po 10 s ~0.01% na ω, ∑P k f32 epsilon. Smazány E3 (broken default) a E3s (mise splněna).

### Censure / Kudos

**Kudos! uživateli (E3 modelshot diagnostika):** Bez explicitního modelshotu z první iterace E3 bych si toho efektu sleep mode nevšiml — pure rapier mode jsem v sezení 3 testoval s G=0 a nulovými velocities (E1/E2), kde sleep nebyl viditelně škodlivý. Joint+pohyb scénář je první případ, kde se to provalilo. Kudos taky za rychlou A1/A2/A3 odpovědi — žádné back-and-forth.

**Kudos! uživateli (zachycení snap při distantním joinu):** Bug s `JointData.fixed(0, 0)` framem by se mohl skrývat dál — pro E3 (oba pixely r=0) byl tichý. Random LMB spawn s nenulovou rotací byl skutečný stress test, kde se chyba projevila viditelně. Default to **adversarial test scenarios** s random initial conditions.

**Censure! sobě:** Predikce E3 baseline („pure rapier baseline pro porovnání s α/β/γ") byla fundamentálně chybná — neuvědomil jsem si, že sleep mode je v default Rapier configu zapnutý a může uspat tělesa s pomalým pohybem. Sezení 2 censure (Rapier integrátor degraduje orbital dynamics) předpovídala obecný problém, ale specificky pro joint scénář jsem nepředvídal sleep jako mechanismus. **Default Rapier configuration audit** by měl být součástí @BEGIN setupu pro každou novou Rapier-based feature.

**Censure! sobě (edge mask iterace):** První návrh `discard` masked edges byl matematicky správný, ale ne vizuálně domyšlený — corner fragmenty (kde dvě hrany jsou stejně blízko) discardly celé rohy, což vytvořilo „černé zobáčky". Sezení 4 mělo censure o **algebraické ověření na nenulovém vstupu**; to platí i pro **vizuální ověření na corner cases**. Při psaní shaderů: vždy mentálně procházet rohy, ne jen středy hran.

**Censure! sobě (joint frame parametr):** `JointData.fixed(anchorA, 0, anchorB, 0)` byl signature lookup, ale nedočetl jsem doc string o významu frame parametrů. Default 0/0 demanduje r_A == r_B, což je sub-problem že to se shoduje s E3 setup (oba r=0) → tichá chyba pro E3, viditelná jen pro random LMB. **API kterého doc je dostupné, číst k odstranění tichého fallthrough.**
